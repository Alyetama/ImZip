Command-line image tool · Rust · single binary

Your images are heavier than they need to be.

imzip fixes that from the terminal. It compresses, resizes and converts images with mozjpeg, oxipng, libwebp and ravif behind one flag set, and it treats batches the way scripts need: parallel work, per-file errors, honest exit codes.

terminal
$ imzip photo.jpg -q 60
photo.jpg -> photo_imzip.jpg ok 182.3 KB -> 4.0 KB

$ imzip shots/ -r --format webp --target-size 30KB

Done: 12 ok, 0 skipped, 0 failed | 11.2 MB -> 349 KB

Install

cargo install --git https://github.com/Alyetama/ImZip

One command, straight from the repo. Needs a Rust toolchain (rustup) and a C toolchain: mozjpeg, libwebp and dav1d compile bundled C. No nasm on aarch64. From a clone instead: cargo install --path .

What it does

One tool, the whole image chore list

Batch without babysitting

Files, folders and glob patterns like **/*.jpg in one command. Work runs across all cores with a progress bar. A corrupt file gets its own line in the report while the rest of the batch finishes.

Compression with real knobs

Quality per format, progressive JPEG, chroma subsampling, PNG palette quantization, lossless WebP, AVIF speed. Or skip the knobs: --target-size 200KB searches the quality setting for you.

Resize any way you measure

Exact dimensions, percentage, max width or height, longest or shortest edge, megapixel cap. Lanczos3 by default. Upscaling stays off unless you pass --allow-upscale.

Convert between five formats

JPEG, PNG, WebP, AVIF and static GIF. Alpha flattens onto a background color when the target format has none. BMP, TIFF and the other read-only formats come out as PNG.

Metadata you decided on

EXIF is stripped and the ICC profile kept by default. Two flags cover the other two philosophies. Details in the metadata section.

Built for scripts

--dry-run prints the plan and writes nothing. --quiet prints errors only. Exit codes split success, partial failure and bad arguments, so CI can react properly.

The three metadata modes

Privacy by default, color accuracy included

EXIF often contains GPS coordinates and camera details you do not want to publish. The ICC profile is what keeps colors right. imzip drops the first and keeps the second unless you say otherwise.

Reference

The flag set, grouped

Input and output
<INPUT>...Files, directories or globs. Directories scan one level deep unless -r.
-r, --recursiveRecurse into input directories.
-o, --output <DIR>Write into DIR, keeping the input structure under it.
--in-placeOverwrite inputs. With a format change, writes a sibling and keeps the original.
--name-template <T>{name} {ext} {format} {width} {height} {index} {parent}. Default: {name}_imzip.{ext}.
--dry-runPrint the plan, write nothing.
--forceOverwrite existing outputs and override both skip rules.
Resize
--width / --height <PX>Both: exact size. One alone: keep the aspect ratio.
--percent <F>Scale to a percentage, e.g. 50 or 33.3.
--max-width / --max-height <PX>Downscale to at most this size. No-op if smaller.
--longest-edge <PX>Downscale so the longest edge equals PX.
--shortest-edge <PX>Scale so the shortest edge equals PX.
--max-megapixels <F>Downscale to at most F megapixels.
--allow-upscalePermit enlarging. Off by default.
--filter <F>nearest, triangle, catmull-rom, lanczos3 (default).
Compression
-q, --quality <0-100>Quality for JPEG, WebP and AVIF. Default: 75.
--progressiveProgressive JPEG.
--chroma-subsampling <444|422|420>JPEG chroma subsampling. Default: 420.
--png-compression <0-6>oxipng optimization preset. Default: 2.
--png-colors <2-256>Quantize to an N-color palette with imagequant, then optimize.
--webp-losslessLossless WebP.
--avif-speed <1-10>ravif speed. 1 is slowest and smallest. Default: 6.
--target-size <SIZE>e.g. 200KB, 1.5MB. Binary-searches quality, keeps the highest that fits. SI units, lossy formats only.
Format and metadata
--format <FMT>jpeg, png, webp, avif or gif. Default: keep input format.
--background <COLOR>Alpha flattening background: #RRGGBB or a name. Default: white.
--strip-all-metadataDrop everything, ICC included.
--keep-metadataCopy ICC, EXIF and XMP into the output, best effort.
Config and diagnostics
--config <PATH>Use this config file. Default: search upward for imzip.toml, .imziprc, .imzip.toml.
-j, --jobs <N>Worker threads: a number, or auto for all cores. Default: auto.
--sequentialOne file at a time, no parallelism. Same as --jobs 1.
-v, --verboseOne line per file, including the chosen quality for size targets.
--quietErrors only, on stderr. -q means quality.
--no-progressDisable the progress bar.

Exit codes

0Everything succeeded, skips included.
1Some files failed; the rest were processed.
2Bad arguments or invalid config.

Config file

Drop imzip.toml in a project root and imzip finds it from any subdirectory. Keys mirror the long flags, CLI values win.

# imzip.toml
quality = 80
format = "webp"
longest_edge = 1600

[jpeg]
progressive = true

imzip is written in Rust. Encoders: mozjpeg, oxipng, imagequant, libwebp, ravif. AVIF decoding by dav1d.
